node_skill_servers module

node_skill_servers.run_node(run_thread=False)[source]

Initialize and run a ROS node for robotic manipulation tasks.

This function sets up a ROS node named ‘manipulation’ using the CustomNode class from pyconnect. It configures the node with agents for various skills (e.g., finding, picking, placing objects) based on provided configurations and modes. The node responds to data inputs by executing predefined skill functions (e.g., find, pick, place) and spins to process messages, either in the main thread or a separate thread depending on the run_thread parameter.

Parameters:

run_thread (bool, optional) – If True, the node spins in a separate thread, allowing the main thread to continue execution. If False, the node spins in the main thread, blocking further execution until the node shuts down. Defaults to False.

Returns:

This function does not return a value. It configures and runs the node as a side effect,

processing messages via the defined response functions.

Return type:

None

Raises:

  • ImportError – If required modules (e.g., pyconnect, skills.recognition) are not installed.

  • KeyError – If expected keys are missing from configuration dictionaries (e.g., SKILL_SERVER_CONFIGS).

  • AttributeError – If agent names or methods are accessed incorrectly due to misconfiguration.

Notes

  • The function uses parse_keys_values to extract runtime arguments from configuration modules like configs.tasks and configs.device_clients.

  • Agents are added to the node from three configuration sets: skill servers, device clients, and show nodes, each with their respective modes (mode, device_mode, show_mode).

  • Response functions are dynamically assigned to agents based on the skill modules (e.g., find, pick) and are executed when the node receives data.
