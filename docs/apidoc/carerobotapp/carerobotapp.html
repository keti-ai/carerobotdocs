carerobotapp.node_prompt module

carerobotapp.node_prompt.run_node()[source]

Run a ROS node for task prompting.

  • Initializes ROS node (task_plansender) with CustomNode.

  • Publishes plans/prompts/actions to agents based on input and config.

  • Spins node in separate thread until ‘exit’ input.

Parameters:

  • isplanned – [True/False]: Decide to send a planned tasks or natural language prompt.

  • play_trajectory – [True/False]: Decide to play the recoded robot state

  • mode – control namespace

Returns:

None

Examples

python3 -m carerobot.node_prompt isplanned=False

carerobotapp.node_taskmanager module

carerobotapp.node_taskmanager.run_node()[source]

ROS node for task manager

  • Initializes ROS node (manager) using CustomNode.

  • Configures task manager agent and agents for skills/devices per settings.

  • Processes task sequence (plan) by sending tasks to relevant agents.

  • Listens for incoming messages indefinitely.

Parameters:

None – This function takes no explicit arguments. It relies on configurations from configs.tasks (e.g., MANAGER_CONFIGS, SKILL_CLIENT_CONFIGS, MODE, DEVICE_MODE) and configs.device_clients (e.g., DEVICE_CLIENT_CONFIGS), parsed via parse_keys_values.

Returns:

None

Raises:

  • ImportError – If required modules (e.g., pyconnect, configs.tasks) are not installed.

  • KeyError – If expected keys are missing from configuration dictionaries or agent responses.

  • NotImplementedError – If a task action is not found in SKILL_CLIENT_CONFIGS or DEVICE_CLIENT_CONFIGS.

  • AttributeError – If agent methods (e.g., send) or configurations are accessed incorrectly.

Notes

  • The taskplan_responsefunc processes a plan split into tasks (format: action::param).
